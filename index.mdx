ApiSquare is a full-stack appointment booking system that lets your clients reserve sessions through a Telegram chatbot while giving you a modern admin dashboard to manage professionals, services, schedules, and reservations — all backed by Vercel KV for production-grade persistence.

## How ApiSquare Works

ApiSquare has two main surfaces that work together:

<CardGroup cols={2}>
  <Card title="Telegram Bot (End Users)" icon="message" href="/bot/conversation-flow">
    Clients open Telegram, type `/start`, and interactively pick a professional, service, date, and time. The bot checks availability in real time and confirms the booking.
  </Card>
  <Card title="Admin Dashboard (Operators)" icon="sliders" href="/admin/overview">
    Operators log in at `/admin`, view upcoming appointments, delete cancelled ones, and configure the professionals' weekly schedules, services, prices, and public holidays.
  </Card>
</CardGroup>

## Get Up and Running

<Steps>
  <Step title="Clone and install dependencies">
    Clone the repository and install Node.js dependencies:

    ```bash
    git clone https://github.com/your-org/apisquare.git
    cd apisquare
    npm install
    ```
  </Step>
  <Step title="Set environment variables">
    Create `.env.local` at the project root and populate your `TELEGRAM_TOKEN`, `ADMIN_PASSWORD`, and Vercel KV credentials (`KV_REST_API_URL`, `KV_REST_API_TOKEN`). See [Environment Variables](/environment-variables) for the full list.
  </Step>
  <Step title="Register the Telegram webhook">
    After deploying to Vercel, run `node setup-webhook.js https://your-app.vercel.app/api/webhook` to point Telegram at your deployment.
  </Step>
  <Step title="Start booking">
    Open your bot in Telegram and send `/start`. Your admin dashboard is live at `/admin`.
  </Step>
</Steps>

<Note>
  ApiSquare stores all reservations in **Vercel KV** (Redis-compatible) in production. When running locally without KV credentials, it transparently falls back to in-memory storage so you can develop without any external dependencies.
</Note>
